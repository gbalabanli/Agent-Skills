# TDD Output Template

Use this structure in final responses.

## Task Type
- `feature` or `bugfix`

## Clarifications and Assumptions
- Clarifications asked
- Assumptions used
- Risk from unresolved ambiguity

## Red: Failing Test
- Test file(s) and case(s) added or updated
- What behavior each failing test specifies
- Failure evidence summary

## Green: Minimal Implementation
- Production files changed
- Minimal logic added or corrected

## Refactor
- Refactors performed with behavior preserved
- Why refactor was safe

## Verification
- Commands run
- Focused test results
- Broader suite results

## Risk and Follow-Up
- Residual risks
- Suggested next tests or hardening steps
